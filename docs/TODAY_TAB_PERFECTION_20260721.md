# Today Tab Perfection Pass

## Product rule

Today must explain CalledOut without onboarding copy:

1. Make a recurring promise.
2. Wait for the fresh-proof window.
3. Submit live proof before the deadline.
4. A miss becomes part of the record and, for circle promises, appears on The Wall.
5. Redemption answers a miss but never erases it.

## Changes

- Replaced technical duration and 24-hour deadline fields with tactile steppers and 12-hour time labels.
- Made recurrence explicit: the schedule repeats on the current weekday until ended.
- Added circle/private audience selection so users understand who holds them accountable.
- Added a fixed fresh-live-photo proof summary and exact proof-open time.
- Added a dark promise summary before confirmation, including the Wall consequence.
- Changed the final action from generic creation to “Lock it in” with a confirmation alert.
- Reworked upcoming cards to show exact proof-open time instead of an ambiguous duration.
- Reduced grace-pass prominence and explained its consequences before use.
- Made commitment headers open the detail screen.
- Added a recurring-schedules screen where users can view or end active schedules.
- Added a secure database function that ends a schedule and removes only future promises whose proof window has not opened.
- Improved Today’s subtitle and empty-state copy to reinforce the commitment loop.
- Made upcoming status pills neutral so the primary proof action has stronger visual hierarchy.
